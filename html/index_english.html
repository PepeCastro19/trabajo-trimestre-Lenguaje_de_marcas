5 Essential HTML Tags and Attributes

1. Tag `<footer>`

Use Case

Defines the footer of a document or section. It typically contains authorship information, copyright data, contact details, or related links.

Example Code


<article>
    <h2>Article Title</h2>
    <p>Content...</p>
    <footer>
        <p>Written by: John Doe</p>
        <p>© 2025 All Rights Reserved</p>
    </footer>
</article>

More Information

W3C: Element `<footer>`

2. Tags `<details>` and `<summary>`

Use Case

Creates an interactive widget that the user can open and close. `<summary>` defines the visible header, and the content inside `<details>` is shown only when expanded.

Example Code


<details>
    <summary>More information...</summary>
    <p>This text is hidden until the user clicks on the summary.</p>
</details>

More Information

W3C: Element `<details>`

3. Tag `<picture>`

Use Case

Allows specifying multiple image sources for different situations (different screen sizes or file formats), improving performance and adaptability (Responsive Design).

Example Code


<picture>
    <source srcset="img-large.jpg" media="(min-width: 800px)">
    <img src="img-small.jpg" alt="Image description">
</picture>

More Information

W3C: Element `<picture>`

4. Attribute `aria-label`

Use Case

Provides an accessible label for elements that do not have visible text on the screen (such as icon buttons). It is fundamental for screen readers to correctly interpret the interface.

Example Code


<!-- Button with search icon, no visible text -->
<button aria-label="Search content">
    <svg>...</svg>
</button>

More Information

W3C: Attribute `aria-label`

5. Tag `<form>`

Use Case

Creates an interactive section to collect user data and send it to a server. It groups elements like text fields, checkboxes, and submit buttons.

Example Code


<form action="/submit" method="POST">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>
    <button type="submit">Submit</button>
</form>

More Information

W3C: Element `<form>`
